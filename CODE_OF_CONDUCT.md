# Code of Conduct

OKoffice follows a contributor-friendly, professional open-source conduct policy.

## Expected behavior

- Be respectful and constructive.
- Assume good intent.
- Focus criticism on code, docs, architecture, and project outcomes.
- Welcome new contributors.
- Credit prior work and related open-source projects.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Discriminatory language.
- Publishing private information.
- Repeated off-topic disruption.
- Abuse of issue trackers for spam or promotion.

## Reporting

Security issues should follow `SECURITY.md`. Conduct issues should be reported to maintainers through the designated private channel once the project is launched.
