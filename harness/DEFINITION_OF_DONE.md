# Definition of Done

A feature is done when:

- It has a public tool name.
- It has typed input/output models.
- It is registered in the tool registry.
- It is callable through CLI if relevant.
- It is callable through MCP/API if stable.
- It returns standard ToolResult.
- It produces artifact manifests for outputs.
- It validates generated PDFs.
- It has tests.
- It has docs and examples.
- It follows dependency/license policy.
