# Governance

AgentPDF Infra should begin with a benevolent-maintainer model and evolve into a small maintainer council as contributors grow.

## Roles

- Contributor: submits issues, docs, tests, or code.
- Reviewer: trusted contributor who reviews PRs.
- Maintainer: can merge PRs and cut releases.
- Security maintainer: handles vulnerability reports.

## Decision process

- Small technical decisions: PR review.
- Public API changes: issue/RFC + maintainer approval.
- License changes: maintainer approval + public notice.
- Major architecture changes: RFC.

## RFC process

Use `community/RFC_TEMPLATE.md`.

## Maintainer responsibilities

- Keep project open-source friendly.
- Protect license safety.
- Maintain docs quality.
- Review security-sensitive tools carefully.
- Keep cloud/proprietary boundary transparent.
